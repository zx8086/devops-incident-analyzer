## What was broken

During SIO-688 pass-4 eval, the elastic sub-agent hit the LangGraph default recursion limit of 25 on the multi-search 5xx-cross-deployment query. Sub-agent ran 202s before failing.

## Trace evidence (the categorization step the original brief mandated)

LangSmith experiment `agent-eval-sio688-pass4-ba31aaf-c3ab89a1`, trace `019e0d35-1e5b-7000-8000-02b04741fb6f`, failing elastic-agent run `019e0d35-461d-76ed-9689-8426ce54ebed`:

- **13 LLM iterations + 12 tools-node executions = 25 graph steps** (= the default recursion limit, exactly).
- **33 underlying `elasticsearch_*` tool calls** under that run (~2.75 per tools-node; the model parallelizes well).
- After the failure, the supervisor's retry path silently re-ran the elastic sub-agent (run `019e0d38-5d9b-...`, 139s, success). User-facing impact: 202s wasted on the failed run *plus* the retry cost.

| Category | Count | Notes |
|---|---|---|
| Distinct refining queries | ~28 | discovery (indices_summary, list_indices x3) -> broad search across logs/apm -> progressive narrowing by index, deployment (`eu_b2c_ecom_api_prd`, `eu_plm_prod`), service (kong, kong-proxy), aggregations (terms, date_histogram), 5xx pivots |
| Near-duplicate pairs | 2-3 | calls 12+13 (identical 398B no-result repeat); 22+23 (terms aggs swapping order) |
| Tool errors eating budget | 2 | call 6: `elasticsearch_get_mappings` 60s timeout; call 11: search ERROR |

Pattern is **legitimate progressive refinement** (cross-deployment 5xx triage), not a tight loop. Improved json-array truncator (PR #45) gives the model enough partial signal to keep refining instead of giving up.

This evidence selected **remediation #1** (raise the limit) over the prompt-budget hint or near-duplicate detection alternatives.

## What shipped (PR #46 -- to be filled in once opened)

**Branch:** `sio-689-elastic-recursion-limit`
**Diff:** 1 file modified + 1 file added, ~50 lines

### `packages/agent/src/sub-agent.ts`
- Added `getSubAgentRecursionLimit(dataSourceId, env?): number | undefined`. Returns 40 for `elastic` (overridable via `SUBAGENT_ELASTIC_RECURSION_LIMIT`); returns `undefined` for everything else so the LangGraph default of 25 stays in force.
- In `runSubAgent`, conditionally spread `recursionLimit` into the `agent.invoke(...)` config (sub-agent.ts:253). Logged on the existing "Invoking sub-agent" info line so eval traces show the budget each run had.

### `packages/agent/src/sub-agent.test.ts` (new)
- Five tests covering: elastic default = 40, non-elastic returns `undefined`, env override honored, invalid values clamp back to 40, fractional values floored.

### Why 40
- 40 / 2 = 20 LLM iterations; at the observed parallelism of ~2.75 tools/iter that's a ~55 tool-call budget vs 33 used in the failing run. Comfortable headroom without runaway latency.
- 5min `SUB_AGENT_TIMEOUT_MS` still bounds wall-clock damage on a true loop.

### Why elastic-only
- Per the original pickup brief: "Don't raise the recursion limit globally. Only the elastic sub-agent showed the problem." Other sub-agents stay on the LangGraph default of 25.

### Why an env var
- Mirrors SIO-686/688 `SUBAGENT_TOOL_RESULT_CAP_BYTES` pattern. Operators can bump without redeploys; supports the eval-driven sizing the SIO-686 handoff established as project norm.

## Gotchas (still load-bearing for next session)

1. **Run eval scripts from workspace root.** Bun's `.env` auto-discovery doesn't walk up from a sub-package cwd. Use `bun --env-file .env packages/agent/src/eval/run-eval.ts`.
2. **`MCP_OAUTH_HEADLESS=true` lives in the MCP server process**, not the eval process. If gitlab/atlassian popups appear during a "headless" eval, restart the MCP server with the flag in *its* env.
3. **`tail -N` buffers stdout.** Pipe straight to a file (`> /tmp/run.log 2>&1`), never through `tail`.
4. **Localhost precheck blocks AgentCore-hosted MCPs.** If kafka/konnect/atlassian aren't running locally, recreate `run-eval-no-precheck.ts` from the snippet in `HANDOFF-2026-05-09-sio-686-eval-sizing.md` (do NOT commit it). This session created it, used it, deleted it.
5. **Bun process hangs post-`Done.` on LangSmith trace upload.** Safe to `kill <pid>` after `Done.` prints -- experiment data is already submitted.
6. **Confidence scores are LLM-judge stochastic.** Single-run deltas are noise; multi-pass averages are signal.
