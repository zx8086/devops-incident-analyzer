### What landed

- `KafkaService.listDlqTopics(options?)` ŌĆö suffix-based DLQ detection (5 patterns), single batched `admin.listOffsets` per topic across all partitions, two-sample `recentDelta` with `Promise.allSettled` parallelization (batch size 20), `null` when second sample fails, topics that fail the first sample are omitted entirely (not phantom-zero entries).
- `AgentState` gains 3 new fields and 3 new types (`AgentName`, `DegradedRule`, `PendingCorrelation`).
- `src/correlation/{rules,engine}.ts` ŌĆö pure rule engine. 4 rules: kafka-empty-or-dead-groups, kafka-significant-lag, kafka-dlq-growth, kafka-tool-failures. All target elastic-agent. Idempotency: rule satisfied if findings already cover triggered entities. Predicate errors fail-open (rule marked satisfied, doesn't break others).
- `src/correlation/enforce-node.ts` ŌĆö `enforceCorrelationsRouter` (returns `Send[] | "enforceCorrelationsAggregate"`), `enforceCorrelationsAggregate` (re-evaluates rules, writes `degradedRules`, caps confidence at 0.6 via `Math.min(currentScore, 0.6)`), `correlationFetch` (one-line wrapper around `queryDataSource`).
- `src/graph.ts` ŌĆö `aggregate -> [correlationFetch | enforceCorrelationsAggregate] -> checkConfidence`. The validate-retry path (validate -> aggregate on retry) re-runs the router; idempotency handles the second pass cleanly.
- 4 c72-style integration tests in `tests/integration/c72-replay.test.ts`: degraded path (asserts no "Elasticsearch not queried" string), satisfied path, no-rules path, plus a contrast sub-test for the empty-group case.

### Spec deviations from `2026-05-07-mandatory-cross-agent-correlation-design.md` (intentional)

1. **Findings shape**: triggers read `state.dataSourceResults` instead of invented `state.kafkaFindings`/`state.elasticFindings`. The plan file already documented this.
2. **Invocation pattern**: routerreturns `Send` objects (not direct call); `correlationFetch` is a thin wrapper around `queryDataSource` exported from `sub-agent.ts` so its output routes to `enforceCorrelationsAggregate` instead of back through `align`/`aggregate`.
3. **Confidence cap honored in aggregate node** (not validate, because validate doesn't touch confidence today).
4. **Retry implicit, not explicit**: rule retry `{ attempts: 3, timeoutMs: 30_000 }` is config-only ŌĆö no explicit retry loop in `correlationFetch`. `queryDataSource` returns error results rather than throwing; idempotency check naturally produces a degraded rule when the elastic findings don't cover the triggered service. This was a deliberate simplification informed by the A0 probe.
5. **DLQ inventory built from scratch** (the spec assumed it existed).
