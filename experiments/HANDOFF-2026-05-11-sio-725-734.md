# Handoff: 2026-05-11 — Kafka MCP pre-production hardening (SIO-725 to SIO-734)

## TL;DR

Before promoting the AgentCore-hosted Kafka MCP to production we audited `packages/mcp-server-kafka/` and filed 10 hardening tickets (SIO-725 to SIO-734). Three are blockers, four are same-milestone follow-ups, three are lower-priority cleanup. The audit lives at `~/.claude/plans/can-we-check-the-functional-flurry.md`.

Trigger: the SIO-716/717/718 incident (c72 false outage from dev/prd misroute) made it clear the MCP server doesn't surface enough signal for the correlation engine to catch the next variation of this bug. These tickets close that gap and harden the AgentCore lifecycle path.

SIO-716 (the env-var fix) is still owned by another team. SIO-717 is **Done** as of 2026-05-11. SIO-718 was already Done.

## Suggested execution order

The order is chosen to make each subsequent ticket cheaper, and to land the highest-incident-prevention value first. Each phase below can be one PR or split — judgment call based on diff size.

### Phase 1 — Service error shape (Wave 1, parallelisable)

**Goal:** the next misroute incident produces a usable signal end-to-end.

1. **SIO-725** (hostname propagation, 4 service files + tests). S.
2. **SIO-729** (content-type guard, same 4 files + new `UpstreamNotJSONError`). S.
3. **SIO-728** (structured `toolErrors` in `response-builder.ts` + agent rule plumbing). M.

These three are tightly coupled — SIO-725 sources the hostname, SIO-729 detects the HTML-503 shape, SIO-728 surfaces both to the correlation engine. Land them in this order in one milestone. SIO-728's agent-side change should keep the three SIO-717 rules green; if their input shape changes, widen them in the same PR.

**Verification at end of Phase 1:** force `KSQL_ENDPOINT=https://ksql.dev.shared-services.eu.pvh.cloud/...` on a c72 query path, run the kafka sub-agent, confirm the LLM transcript contains `Kafka Connect (connect.dev.shared-services.eu.pvh.cloud) upstream returned text/html 503` (not nginx HTML), and confirm the `infra-service-degraded-needs-synthetic-cross-check` rule fires in LangSmith. This is the regression test for the SIO-716 scenario.

### Phase 2 — AgentCore lifecycle (Wave 2, parallelisable)

**Goal:** the MCP behaves correctly through AgentCore deploys and health probes.

4. **SIO-726** (`/ready` endpoint with 30s probe cache).
5. **SIO-727** (graceful shutdown drain on SIGTERM).

Independent of Phase 1 and of each other. SIO-727 wants verification of what AgentCore actually sends as a stop signal — check with the AgentCore-owning team or read the framework docs before assuming SIGTERM.

**Verification at end of Phase 2:** locally start the server, hit `/ready` while one upstream is unreachable (assert 503), restart the upstream, hit `/ready` again (assert 200). Local SIGTERM test: start a long `consume_messages` call, send SIGTERM, confirm the call completes before exit.

### Phase 3 — LLM signal quality

**Goal:** the agent calls the right tool with the right scope.

6. **SIO-730** ([READ]/[WRITE]/[DESTRUCTIVE] tags).
7. **SIO-731** (paginate `list_topics`).
8. **SIO-732** (gate destructive tools at registration).

Independent of Phases 1-2. SIO-732 has a small dependency on SIO-730 if you want the lint test for tag prefixes to also cover the dropped tools (you can register the test against the registered set, not the source set, so this isn't a hard dep).

### Phase 4 — Hardening backlog

9. **SIO-733** (AgentCore round-trip integration test). L.
10. **SIO-734** (ephemeral consumer cleanup). S.

SIO-733 is the largest single item — full mock plumbing for `BedrockAgentRuntimeClient`. Plan it standalone; don't bundle. SIO-734 is small and unblocks nothing else.

## Dependencies and gotchas

- **SIO-725 + SIO-728 are coupled.** The hostname plumbed by SIO-725 lands in `toolErrors[].hostname` only if SIO-728 is in the same milestone. If they ship apart, document the in-flight shape.
- **SIO-728 touches `packages/agent/src/correlation/rules.ts`.** The three SIO-717 rules read `result.toolErrors` today (per SIO-717 design). Before adding new shape, verify the rules actually parse it correctly in the existing test fixtures.
- **SIO-727 SIGTERM assumption needs validation.** AgentCore might send a framework-specific stop hook or SIGKILL. Don't burn cycles writing the drain logic against the wrong signal — check first.
- **SIO-731 changes a tool signature.** Anyone calling `list_topics` from outside the agent (manual MCP debug sessions, the SigV4 proxy bash one-liners per the `feedback_probe_agentcore_via_sigv4_proxy` memory) will need to know about the new `limit`/`prefix` params and the truncation marker. Update the memory note when shipping.
- **`bun --hot` does NOT re-resolve node_modules** (per `reference_bun_hot_does_not_reresolve_modules` memory). Full restart needed between dependency-affecting changes.
- **A1 from the audit (admin-init race) is NOT a bug.** The SIO-710 pattern in `client-manager.ts:84-104` is correct. Don't file or "fix" it — verified during this audit.
