## Gotchas and notes from this session

### Hard blockers to watch

1. **Tab indentation.** Repo uses tabs. Plan shows 4-space blocks. Convert on paste or Biome will rewrite on save.

2. **`bun test` path sensitivity.** Running `bun test packages/...` from outside the worktree silently targets the wrong tree. Always start commands with `pwd` or `cd` into the worktree. This burned a minute this session.

3. **`mock.module` is process-wide in Bun.** This was the big lesson from SIO-640: any `mock.module("./some-sibling.ts", ...)` leaks across every test file in the same `bun test` run. The SIO-641 plan is written as a single test file with no mock.module at all — which is good, just don't "improve" it by adding sibling mocks.

4. **`supervisor-router.test.ts` already mocks `./prompt-context.ts`**, which leaks into any other test file importing the real prompt-context module. SIO-641 doesn't touch prompt-context so this shouldn't matter, but if you see unexpected behavior in the full test run vs. single-file run, check for pollution.

5. **`validation.test.ts` builds the real graph via `buildGraph()`.** When the full agent test suite runs after SIO-641 lands, this graph-boot will log `"Runbook selector node enabled"` or `"disabled"` depending on whether the mock pollution from supervisor-router flipped `agent.runbookSelection` to undefined mid-run. Harmless but noisy in output; don't let it confuse you.
