## What was broken

Per the SIO-690 handoff: during the first eval run on 2026-05-09, every gitlab sub-agent failed with `allToolsFailed: true` because `MCP_OAUTH_HEADLESS` was unset under `bun run dev`. The TTY auto-detect in `packages/shared/src/oauth/headless.ts:6` only fires when stdout is not a TTY -- and `bun run dev` in a terminal IS a TTY -- so OAuth ran in interactive mode and the gitlab MCP looped on browser popups. ~$5-8 of LLM cost and ~10 min wall-clock burned before the misconfig was caught (popup loop is a UI-only signal; eval logs only show tool errors, not why).

Root cause confirmed in code:

- `packages/shared/src/oauth/headless.ts:6` — `MCP_OAUTH_HEADLESS === "true" || process.stdout.isTTY === false`. Auto-detect misses TTY dev sessions.
- `.env.example:127` had `MCP_OAUTH_HEADLESS=` (empty), which means anyone copying `.env.example -> .env` got the unsafe default.
- No boot-time check anywhere -- the misconfig was only observable after the first OAuth-requiring tool call.

### `.env.example` (the headline change)

New default `MCP_OAUTH_HEADLESS=true` with a comment block that leads with the value, names both seed commands inline, and drops the now-misleading "Auto-detected when stdout is not a TTY" line. Existing `.env` files are unchanged -- this only affects future `cp .env.example .env` users, who now get the safe default.

### Helpers (new, `packages/shared/src/oauth/`)

`seeded-tokens.ts` (~22 lines) — exports `hasSeededTokens(namespace, key): boolean`. Reads `~/.mcp-auth/{namespace}/{sanitized_key}.json`, returns `true` iff the file exists, parses as JSON, and `parsed.tokens.access_token` is a non-empty string. No mkdir, no logger, IO/parse errors return `false` silently. Imports `getOAuthStoragePath` from `./base-provider.ts` directly (does not route through the `oauth/index.ts` barrel — keeps `getOAuthStoragePath` internal-to-shared).

`boot-warn.ts` (~55 lines) — exports `warnIfOAuthNotSeeded(options)` with `hasSeededTokensFn` / `isHeadlessFn` test seams. Behavior:

- `!seeded && !headless`: emit a `!!!`-prefixed ALL-CAPS WARN.
- `headless`: emit one INFO line with `{ namespace, seeded: <bool> }`.
- `seeded && !headless`: silent (happy path).

### Refactor (`packages/shared/src/oauth/base-provider.ts`)

Extracted `getOAuthStoragePath(ns, key)` as a new export (no mkdir). The existing `getStoragePath` is now a thin `ensureStorageDir + getOAuthStoragePath` wrapper. This was load-bearing: it prevents the new presence-checker from drifting away from the writer's sanitization regex, which is the exact bug class we were trying to prevent. The external contract of `BaseOAuthClientProvider.saveTokens` / `loadState` is preserved.

### Wiring

Both gitlab and atlassian MCP entrypoints (`packages/mcp-server-{gitlab,atlassian}/src/index.ts`) call `warnIfOAuthNotSeeded` after the "Starting ... MCP Server" log and before `proxy.connect()`, so the operator sees the misconfig at boot rather than after the eval starts failing:

```ts
warnIfOAuthNotSeeded({
  namespace: "gitlab",  // or "atlassian"
  key: config.gitlab.instanceUrl,  // or config.atlassian.mcpEndpoint
  endpointLabel: "instanceUrl",  // or "mcpEndpoint"
  seedCommand: "bun run oauth:seed:gitlab",  // or :atlassian
  logger: serverLog,
});
```

### Barrel exports

`packages/shared/src/oauth/index.ts` adds `hasSeededTokens` and `warnIfOAuthNotSeeded` (+ type) re-exports. `packages/shared/src/index.ts` (top-level barrel) re-exports the same. Both are now part of the `@devops-agent/shared` public API.

### WARN format

```
!!! GITLAB OAUTH NOT SEEDED AND MCP_OAUTH_HEADLESS IS NOT SET --
first request will pop a browser. Run `bun run oauth:seed:gitlab` to seed
tokens, OR set MCP_OAUTH_HEADLESS=true in .env to fail fast on auth.
```

Structured metadata: `{ namespace, [endpointLabel]: <value>, remediation: <seedCommand>, docs: ".env.example MCP_OAUTH_HEADLESS" }`. The `!!!` + ALL-CAPS prefix is deliberate — SIO-688/689/690 sessions all hit "didn't notice the warn" failure modes, and standard pino lines blur together in dev tail.

### Why this design

- **Default-on in `.env.example`, not in `bun run dev`.** Per ticket — option (2) "default-on in `bun run dev`" was rejected because it surprises CLI seeders and stand-alone users. `.env.example` is the documented contract.
- **Boot-time WARN, not a `proxy.connect()` retry suppression.** The gitlab MCP's existing `try/catch` around `proxy.connect()` (`packages/mcp-server-gitlab/src/index.ts:47-50`, untouched) already handles connect failures — but it fires only after the OAuth flow has been attempted. The boot-time WARN fires *before* `proxy.connect()`, so operators see the misconfig immediately.
- **Read the storage file, don't construct the provider.** `BaseOAuthClientProvider.hasValidTokens()` would tell us the same thing, but constructing the provider just to ask requires the full DI chain (callback port, redirect handler, etc.) and would be visible as an extra log line at boot. A bare file-existence check via `getOAuthStoragePath` is simpler and cheaper.
- **Test seams over module mocking.** `hasSeededTokensFn` / `isHeadlessFn` parameters in `warnIfOAuthNotSeeded` keep the unit tests independent of the filesystem and env, while the production callers (gitlab, atlassian) use the defaults and exercise the full code path.

### Out of scope (not pulled in, per ticket)

- Did not change `bun run dev` to inject `MCP_OAUTH_HEADLESS=true` (option 2 — rejected by ticket).
- Did not add a header-propagation auth signal (option 3 — rejected by ticket).
- Did not replace the OAuth seeder workflow.
- Did not add a "dev mode" that bypasses auth.
- Did not touch the TTY auto-detect in `headless.ts:6` — left in place as the non-TTY safety net for CI / piped contexts.

## Gotchas (carry forward)

Mostly inherited from SIO-686 / SIO-688 / SIO-689 / SIO-690 sessions. New gotchas from this session marked **NEW**.

1. **NEW: bun workspace resolution doesn't work for ad-hoc scripts at the repo root.** The one-shot E2E verification script needed to live inside `packages/shared/src/` and import from relative paths (e.g. `./oauth/seeded-tokens.ts`) rather than `@devops-agent/shared`. A script at the repo root or in `/tmp/` will fail with "Cannot find module '@devops-agent/shared'". For future session-local verification scripts, place them under a workspace package's `src/` and use relative imports.

2. **NEW: biome's `useSortedAttributes` rule sorts type imports BEFORE value imports alphabetically.** When adding a value + type pair to a multi-import block, put the `type Foo` line *before* the `foo` value line within the block, even if alphabetical ordering would suggest otherwise. Biome flagged this on the first commit attempt against `packages/shared/src/index.ts`.

3. **NEW: `noUncheckedIndexedAccess` strict mode flags `const [foo] = array` destructuring** when the array length isn't statically known. Test files in `packages/shared` use the pattern `const call = logger.calls[0]; if (!call) throw new Error(...)` instead of array destructuring. Mirror this for any new test that pulls a single element from a `.calls[]` array.

4. **Run eval scripts from workspace root.** Bun's `.env` auto-discovery doesn't walk up from a sub-package cwd. Use `bun --env-file .env packages/agent/src/eval/run-eval.ts`. (Inherited.)

5. **Localhost precheck blocks AgentCore-hosted MCPs.** Konnect is AgentCore-only and not bound locally; precheck would fail. Recreate `run-eval-no-precheck.ts` in `packages/agent/src/eval/` from the `run-eval.ts` template (drop the `spawnSync` precheck call); **do NOT commit it**. (Inherited.)

6. **Bun process hangs post-`Done.` on LangSmith trace upload.** Safe to `kill <pid>` after `Done.` prints -- experiment data is already submitted. (Inherited.)

7. **`tail -N` buffers stdout.** Pipe straight to a file (`> /tmp/run.log 2>&1`) or use `Monitor` with `tail -f | grep --line-buffered`. (Inherited.)

8. **`MCP_OAUTH_HEADLESS=true` lives in the MCP server process, not the eval process.** If gitlab/atlassian popups appear during a "headless" eval, restart the MCP server (or `bun run dev`) with the flag in *its* env. SIO-690 learned this the hard way; SIO-693 makes it default-on but the principle is unchanged. (Inherited; the SIO-693 fix also adds a boot-time WARN if you forget.)

9. **`bun run dev` runs `--filter '*' dev` across all workspace packages.** Some packages bind ports (couchbase :9082, gitlab :9084, atlassian :9085, elastic :9080), others bind to non-default ports (kafka via AgentCore proxy on :3000), konnect doesn't bind locally at all. (Inherited.)

10. **`langsmith experiment get` returns aggregate only.** For per-query feedback comments, use `langsmith run list --project <experiment-name> --name runAgent --full --format json` and parse `feedback_stats` per-run. (Inherited.)

11. **Trace categorization for sub-agent runs requires the langsmith Python SDK,** not just `langsmith-fetch` CLI. The CLI's `traces` command pulls top-level eval-judge runs; child runs (where the actual sub-agent error and tool-call breakdown live) need `client.list_runs(project_id=..., trace_id=...)` paginated. Use the venv at `/Users/Simon.Owusu@Tommy.com/.local/pipx/venvs/langsmith-fetch/bin/python` -- it already has the SDK. (Inherited.)
